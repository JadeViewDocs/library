# JadeView GitHub Issues 模板创建计划

## 1. 项目分析
JadeView 是一个基于 Rust 开发的 WebView 窗口库，提供 C 语言兼容的 API 接口，方便在其他语言（如易语言）中调用和使用。

## 2. 模板创建目标
创建以下 GitHub Issues 模板，帮助用户规范地提交问题和请求：

### 2.1 Bug 报告模板
- **文件位置**：`.github/ISSUE_TEMPLATE/bug_report.yml`
- **功能**：规范 Bug 报告的格式，确保包含足够的信息
- **包含内容**：
  - 问题描述
  - 预期行为
  - 实际行为
  - 复现步骤
  - 环境信息（系统版本、JadeView 版本等）
  - 日志和截图

### 2.2 功能请求模板
- **文件位置**：`.github/ISSUE_TEMPLATE/feature_request.yml`
- **功能**：规范功能请求的格式，确保包含清晰的需求描述
- **包含内容**：
  - 功能描述
  - 动机和使用场景
  - 建议的实现方式
  - 替代方案
  - 附加信息

### 2.3 问题咨询模板
- **文件位置**：`.github/ISSUE_TEMPLATE/question.yml`
- **功能**：规范问题咨询的格式，帮助用户获得更好的解答
- **包含内容**：
  - 问题描述
  - 尝试过的解决方法
  - 相关代码或截图
  - 环境信息

### 2.4 贡献模板
- **文件位置**：`.github/CONTRIBUTING.md`
- **功能**：指导开发者如何为项目做贡献
- **包含内容**：
  - 开发环境搭建
  - 代码规范
  - 提交 PR 的流程
  - 测试要求

### 2.5 自定义配置模板
- **文件位置**：`.github/ISSUE_TEMPLATE/config.yml`
- **功能**：配置 Issues 模板的默认行为

## 3. 实施步骤
1. 创建 `.github/ISSUE_TEMPLATE/` 目录
2. 编写每个模板文件的内容
3. 确保模板符合 GitHub 规范
4. 添加详细的说明和示例
5. 测试模板的有效性

## 4. 预期效果
通过这些模板，用户可以更规范地提交 Issues，开发者可以更快地理解和解决问题，提高项目的维护效率。
